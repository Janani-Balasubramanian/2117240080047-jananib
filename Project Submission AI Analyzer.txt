Project Submission AI Analyzer
Overview
Build a Python service from scratch that analyzes a student's project submission (ZIP file) and
produces an evaluation package for a mentor-led viva.
Given only project metadata and a ZIP, the service should:
1. Extract and analyze the codebase
2. Suggest relevant skills from a provided catalog
3. Generate interview questions per skill
4. Return a structured evaluation report (JSON)
Problem
Universities assess experiential projects (internships, capstones, mini-apps). Mentors need help
preparing viva questions that reflect what the student actually built — not generic textbook questions.
Input: ZIP file + project metadata + skill catalog (JSON we provide)
Output: ranked skills + skill-wise questions + evaluation report
Required flow
ZIP + metadata + skill catalog
↓
[1] Safe ZIP extraction → file tree, dependencies, code snippets
↓
[2] Suggest skills (only from catalog, with confidence + rationale)
↓
[3] Generate interview questions per suggested skill and summary
↓
[4] Return structured evaluation report (JSON)
What to build
API
One endpoint:
POST /analyze-submission
Input (multipart form):

project_title string yes
project_description string no
project_outcomes string yes
zip_file file yes
questions_per_skill int no (default: 2)

Technical requirements
Item Requirement
Language Python 3.11+
Framework FastAPI
LLM Use own AI Key
Data No database — skill catalog comes in the file you save
Questions Per skill: at least 1 conceptual + 1 codebase_specific (must cite real file paths or
symbols from the ZIP)
Errors Clear HTTP responses for bad ZIP, empty project, invalid catalog, LLM failures
Config .env.example for API keys — no secrets in submission
What we provide
Shared Git repository (access will be given to you)
Sample skill catalog JSON
Optional: 1–2 example ZIP submissions for testing
Everything else — project structure, ZIP parsing, prompts, API, tests — you build from scratch.
Deliverables
1. Complete runnable project — install deps + one command to start the server
2. README — setup, env vars, folder structure, how to run tests, one example API call, and demo
video link
3. Your own test ZIP — small project you zipped for demo (5–15 source files)
4. 2+ unit tests (optional) — e.g. path-traversal ZIP rejected; output matches your Pydantic schema
5. Short write-up (optional) (½ page) in README or NOTES.md
Submission
1. Git branch (required)
Push your code to the shared repository we provide. Do not commit directly to main or master .
Branch naming convention:
<college-id>-<full-name-in-lowercase-with-hyphens>
Rule Good Bad
College ID first, then name 21cs0123-arun-kumar arun-kumar-21cs0123
All lowercase 22it0456-priya-sharma 22IT0456-Priya-Sharma
Hyphens between words 21ec0789-r-venkatesh 21ec0789_r_venkatesh
No spaces or special characters 20cs1001-john-doe 20cs1001 john doe
Steps:
git clone <shared-repo-url>
cd <repo-name>
git checkout -b 21cs0123-arun-kumar
# ... add your code ...
git add .
git commit -m "Add project submission analyzer"
git push -u origin 21cs0123-arun-kumar
Do not commit: API keys, .env files, or secrets. Use .env.example only.
